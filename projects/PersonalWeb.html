Old design

I originally started building this website during my second year of university, as I realised that an online portfolio could showcase my skills better than a simple CV.

However, the original design, hastily done in less than an afternoon, was quite bad to say the least. The weird tone of green strained the eyes, it wasn't mobile friendly, and lacked a general vision. Overall, it was in clear need of a redesign. As I was occupied with school work, I had to leave this for the summer.

Planning

Before coding the website, I started by designing the branding. I selected easy to read sans-serif fonts that complemented each other well: P22 Underground for the titles, and Brandon Grotesque for the paragraphs. Concerning colours, I utilized tones that had a strong personal significance: this website is supposed to represent me, after all. The main green colour has a double meaning: it's both the colour of my eyes and the one of the mountains that cover the region my family originates from.

After I had a branding I was happy with, it was time to start prototyping. I used Adobe XD to make interactive mockups, as I already had experience using it for another of my projects, EventLite. By making both desktop and mobile artboard, I had a clear idea of what to develop and how my website should look like on all viewports.

There are plenty of small details meant to give the website a bit more personality. The accent colour in the project pages changes to that of the project itself, links to third party products take the colour of said product's branding (or an accessible close-match).

Development

I used Visual Studio Code as my editor, as I also wanted to use tools I was not familiar with during this project. Thanks to GitHub's issue and project features, it was easy to plan out the next steps to take in development.

This website is hosted using GitHub Pages, which poses a small problem: it must be a static site. This means that I can't use PHP code for some of the planned features, like the contact form. However, I can't justify the cost of hosting the website elsewhere for a small number of visits. Creating my own hosting server has its own complications too, as the website would be down everytime I travel to university or back home. I'm currently researching 3rd party alternatives that can be used in a static site.

To ensure that the website is accessible, I used the WAVE tool, fixing contrast errors and alerts. I also used Lighthouse to check that there weren't any issues with performance.
